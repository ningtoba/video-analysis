## Executive Summary

The platform is remarkably mature — 40 versions in ~24 hours, every major feature implemented. Research across 5 domains combined with deep codebase analysis yields 3 high-impact feasible features and confirms the REST API layer as the top priority.

## 1. Video MLLM Breakthroughs (2025-2026)

### InternVideo3 — June 10, 2026 (arXiv:2606.12195) — MOST RELEVANT NEW MODEL
- Multimodal Contextual Reasoning (MCR): closed-loop long-video understanding as evidence accumulation
- M^2LA: token-preserving KV-cache compression (relevant optimization technique)
- No small/quantized variants available for 12 GB VRAM yet — monitor only

### VideoThinker-R1-3B — CVPR 2026 (April 2026)
- 3B params (~6-7 GB BF16), MIT license, based on Qwen2.5-VL-3B
- Surpasses VideoChat-Flash by +7% on Video-MME
- Medium-term candidate — fits 12 GB when VRAM freed

### Verdict: No replacement needed now
Current dual-model (SmolVLM2 2.2B + Qwen3-VL-30B-A3B FP8) + LLMProvider covers full 2-8 GB VRAM spectrum. Models searched: VideoLLaMA3, VideoMamba, InternVideo2.5, LLaVA-Video-7B, Video-Scan, VideoKR — none justify replacement on 12 GB hardware.

## 4. Video Quality Assessment

### ALREADY FULLY IMPLEMENTED (quality.py)
- Blur detection (Laplacian variance, configurable threshold)
- Brightness check (mean pixel, too dark / too bright)
- Static frame detection (SSIM-style between consecutive frames)
- Corruption check (FFprobe decode errors)
- Quality-based stage skipping (skip OCR when blurry, skip YOLO when dark/bright)
- All CPU-based, <1ms per frame, zero VRAM
- Integrated in pipeline, results stored in FrameInfo.metadata

### Minor Enhancement
- Noise detection (30 min, invert Laplacian variance threshold)
- Enable CLIP frame dedup (already configurable, needs threshold tuning)

## 5. Agentic Multi-Video Workflows

Already has: Agentic Video Agent (7 tools, v0.36.0) | Federated Search | Multi-hop decomposition | Gradio 6 Workflow

### Gap: Cross-Video Identity Tracking (Highest Impact)
InsightFace generates 512-d face embeddings per-video but no cross-video identity matching.
Add: face embedding ChromaDB index -> persistent identity across videos.
- Feasibility: 1-2 days, reuses existing InsightFace + ChromaDB
- Impact: High — enables "find this person across all my videos" queries

### Gap: Chained Workflows
"Find all videos with person X then transcribe", "Detect cars then OCR plates" — natural agent evolution.

## Conclusion

Three most impactful next features:
1. REST API Layer + OpenAPI docs (P0) — wraps existing backend methods, opens platform to external consumption. Infrastructure ready, gap is routing + docs exposure.
2. Cross-video identity tracking (P1) — InsightFace embeddings + ChromaDB for cross-video person queries.
3. Async job API (P2) — natural follow-up to REST layer for long pipeline runs.
