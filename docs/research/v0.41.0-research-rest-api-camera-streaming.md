## 1. Full REST API Layer

### Current State

The platform had a minimal FastAPI health app (`ui/health.py`) with only:
- `GET /health` — container health check
- `GET /api/library` — list indexed videos
- `GET /api/video/{id}` — single video info
- `GET /metrics` — Prometheus metrics
- `GET /api/federated/search` — federation peer endpoint

### Design

Created `video_analysis/api.py` with a factory function `create_api_router()`
returning a FastAPI `APIRouter`. Key design decisions:

- **Pydantic v2 models** for all request/response schemas — auto OpenAPI docs
- **SSE streaming** via `StreamingResponse` + `asyncio.Queue` pattern for
  real-time token delivery
- **Async endpoints** with `run_in_executor` for blocking pipeline/RAG calls
- **Consistent error responses** — `ErrorResponse` model for all errors
- **Module-level RAG instance** shared with `ui/health.py`

### Endpoints

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/videos/process` | POST | Submit URL/file for pipeline processing |
| `/api/videos` | GET | List all indexed videos |
| `/api/videos/{video_id}` | GET | Full video details |
| `/api/videos/{video_id}` | DELETE | Remove video from index |
| `/api/videos/{video_id}/query` | POST | Ask question, get answer |
| `/api/videos/{video_id}/query/stream` | POST | SSE streaming answer |
| `/api/videos/search` | GET | Cross-video semantic search |
| `/api/videos/{video_id}/transcript` | GET | Full transcript |
| `/api/videos/{video_id}/frames/{timestamp}` | GET | Frame JPEG image |
| `/api/videos/{video_id}/chapters` | GET | Auto-generated chapters |

### OpenAPI

FastAPI auto-generates full OpenAPI 3.1 specification at `/openapi.json`
with interactive Swagger UI at `/docs` and ReDoc at `/redoc`.

## 2. MLLM Streaming Q&A

### Current State

Both `HermesProvider` and `OpenAIProvider` only supported synchronous
`chat()` and `structured_chat()` — the full response was returned at once.

### Implementation

- **`stream_chat()` abstract method** added to `LLMProvider` ABC:
  - `HermesProvider.stream_chat()` — uses `asyncio.create_subprocess_exec`
    with `--stream` flag, falls back to word-by-word emission from
    synchronous `chat()` when subprocess mode fails
  - `OpenAIProvider.stream_chat()` — uses `httpx.AsyncClient` with
    `stream=True` and parses SSE `data:` events for `delta.content` tokens
  - Proper error handling: graceful fallbacks for timeout, FileNotFoundError,
    and API errors

- **`StreamChatManager`** class:
  - Session lifecycle (create, get, end, evict stale)
  - Conversation history per session
  - Configurable max concurrent sessions (default 50)
  - Async generator-based `stream()` method

- **`stream_llm_response()`** convenience function — wraps provider directly

## 3. Webcam / Live Camera Capture

### Design

Created `ui/camera.py` as a self-contained Gradio 6 tab that:

- Uses `gr.Image(sources=['webcam'])` for browser-based webcam access
- Falls back to `gr.Image(sources=['upload'])` when webcam is unavailable
- Captures frames and runs lightweight analysis via existing YOLO + OpenCLIP
- Supports continuous auto-analysis mode via JS interval timer
- Shows structured results: detected objects with confidence, CLIP scene label
- Config toggle via `CAMERA_ENABLED` env var

### Architectural Decisions

- **No new heavy dependencies** — reuses existing YOLO and OpenCLIP models
- **Lazy Gradio imports** — the module is importable without gradio installed
- **Temp file management** — captured frames saved to temp files, cleaned up
- **Graceful degradation** — when webcam is unavailable, upload mode works
