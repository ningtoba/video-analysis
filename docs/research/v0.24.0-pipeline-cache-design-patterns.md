## Executive Summary

A `PipelineCache` class for the video-analysis pipeline is planned that:

- **Content-addressable**: SHA-256 hash of (video content + stage name + config params)
- **Two-level storage**: Manifests (stage→key mapping) and blobs (serialized outputs)
- **In-place integration**: Minimal changes to `pipeline.py` — stages become cache-aware
- **Three invalidation modes**: Explicit `--force`, automatic (config hash mismatch), TTL-based

Estimated impact from the research document:
- Full re-run → ~5s (hash check only)
- Partial re-run (config change) → ~2min (only affected stages)
- Add new stage → ~30s (previously cached stages reused)

## Patterns from Open-Source Projects

### Pattern A: DVC (Data Version Control) — Content-Addressable Blob Store

**Source:** [dvc.org](https://dvc.org) — 15k+ GitHub stars

**Core mechanism:**
- Each file/directory gets a `.dvc` file with its MD5/SHA256 hash
- Files are stored in `.dvc/cache/files/md5/` (two-level directory: first 2 chars as dir, rest as filename)
- Cache is shared via remote storage (S3, GCS, etc.)
- Pipeline stages (dvc.yaml) track dependency hashes; if any dep hash changes, the stage re-runs

```python
# Pseudo-code of DVC's cache mechanism
import hashlib
from pathlib import Path

CACHE_DIR = ".dvc/cache"

def _hash_file(path: Path) -> str:
    h = hashlib.sha256()
    with open(path, "rb") as f:
        for chunk in iter(lambda: f.read(65536), b""):
            h.update(chunk)
    return h.hexdigest()

def cache_path(content_hash: str) -> Path:
    return CACHE_DIR / "files" / content_hash[:2] / content_hash[2:]

def cache_put(source: Path, content_hash: str) -> Path:
    dest = cache_path(content_hash)
    dest.parent.mkdir(parents=True, exist_ok=True)
    if not dest.exists():
        shutil.copy2(source, dest)
    return dest

def cache_get(content_hash: str) -> Optional[Path]:
    dest = cache_path(content_hash)
    return dest if dest.exists() else None
```

**Key lessons for PipelineCache:**
- Two-level directory hashing (first 2 chars) prevents millions of files in one directory
- Hash is computed from file *content*, not filename — truly content-addressable
- Remote cache + local cache = cache hierarchy
- Pipeline re-runs only when dependency hashes change (DAG-based invalidation)

### Pattern B: joblib.Memory — Argument Hashing + Pickle Cache

**Source:** [joblib.readthedocs.io](https://joblib.readthedocs.io/en/stable/memory.html) — Used by scikit-learn

**Core mechanism:**
- Decorator `@memory.cache` on any function
- Computes a hash of (function_name, input_args, input_kwargs)
- Stores output as `.pkl` in `location/<func_name>/` directory
- On subsequent call, if hash matches and cache file exists, loads pickle

```python
from joblib import Memory
memory = Memory(location="./cache", verbose=0)

@memory.cache
def train_model(X, y, params):
    # Expensive computation
    return model
```

**Hash computation (simplified from joblib source):**
```python
import hashlib
import pickle

def _compute_func_hash(func, args, kwargs):
    h = hashlib.sha256()
    # Hash function bytecode
    h.update(func.__code__.co_code)
    # Hash arguments
    h.update(pickle.dumps((args, sorted(kwargs.items())), protocol=5))
    return h.hexdigest()
```

**Key lessons for PipelineCache:**
- Function + arguments = cache key (simple, effective)
- Pickle is fine for serialization but has security considerations
- Cache per-function (ours: per-stage) keeps invalidation granular

### Pattern C: Hugging Face Datasets — Cache with Signature Hashing

**Source:** `datasets` library — Used by HuggingFace ecosystem

**Core mechanism:**
- Each dataset transformation is cached by its "signature" — a hash of the transformation config
- Cache stored in `~/.cache/huggingface/datasets/`
- Cache invalidation when version/parameters change

```python
# Simplified from datasets library
class CacheManager:
    def __init__(self, cache_dir="~/.cache"):
        self.cache_dir = Path(cache_dir).expanduser()

    def get_cache_path(self, func_name: str, kwargs: dict) -> Path:
        sig = self._compute_signature(func_name, kwargs)
        return self.cache_dir / func_name / f"{sig}.arrow"

    def _compute_signature(self, func_name: str, kwargs: dict) -> str:
        import hashlib, json
        # Normalize kwargs to deterministic JSON
        normalized = json.dumps(kwargs, sort_keys=True, default=str)
        return hashlib.sha256(
            f"{func_name}:{normalized}".encode()
        ).hexdigest()
```

**Key lessons for PipelineCache:**
- JSON-normalized config parameters make hash deterministic
- `.arrow` (Apache Arrow) for large tabular data; we use `.pkl` for structured Python objects
- Cache versioning: embedded format version in cache key to handle schema migrations

### Pattern D: MLflow / Kubeflow Pipelines — Per-step Artifact Cache

**Source:** MLflow Pipelines / Kubeflow Pipelines SDK

**Core mechanism:**
- MLflow "steps" each produce artifacts stored in `mlruns/`
- Each step run is identified by a run ID + step name
- Artifacts are content-addressable via MLflow's artifact store

```python
# MLflow Pipelines pattern
class PipelineStep:
    def run(self, inputs: dict) -> dict:
        cache_key = self._compute_cache_key(inputs)
        cached = self._load_from_cache(cache_key)
        if cached is not None:
            return cached
        outputs = self._execute(inputs)
        self._save_to_cache(cache_key, outputs)
        return outputs
```

**Key lessons for PipelineCache:**
- Each stage implements a consistent `run(inputs) -> outputs` interface
- Cache check is the FIRST thing in `run()` — before any model loading
- Cache key includes: stage name, input hashes, config version, code version

### Pattern E: Snakemake / Luigi — DAG-aware Pipeline Caching

**Source:** Snakemake (bioinformatics pipelines) — 2k+ GitHub stars

**Core mechanism:**
- Each rule has `input`, `output`, `params` — cache key = hash of all three
- If output files exist and hash matches, rule is skipped
- DAG resolution means only changed rules + downstream rules re-run

```python
# Snakemake-style rule
rule transcribe:
    input: "audio/{video_id}.wav"
    output: "transcripts/{video_id}.json"
    params: model="large-v3"
    shell: "whisper {input} --model {params.model} -o {output}"

# Cache key (implied):
# hash(input_files + params + rule_name + tool_version)
```

**Key lessons for PipelineCache:**
- Output file existence is NOT enough — must also check hash of inputs + params
- Downstream dependency tracking: if stage A changes, stages B, C that depend on A must also re-run
- Our pipeline is linear (DAG is a chain), so downstream tracking is straightforward

## Recommended Architecture for PipelineCache

### Two-Level Cache Structure

```
data/
└── cache/
    ├── manifests/           # Stage → blob key mapping (one JSON per video)
    │   └── <video_hash>.json
    └── blobs/               # Actual serialized stage outputs (one pickle per stage)
        └── <content_hash>.pkl
```

**Manifest JSON structure** (`data/cache/manifests/<video_hash>.json`):
```json
{
  "version": 1,
  "video_hash": "abc123...",
  "created_at": "2026-06-26T22:18:00Z",
  "stages": {
    "scene_detection": {
      "config_hash": "def456...",
      "blob_hash": "7890ab...",
      "cached_at": "2026-06-26T22:18:00Z",
      "duration_ms": 1234
    },
    "transcribe": {
      "config_hash": "ghi789...",
      "blob_hash": "cdef01...",
      "cached_at": "2026-06-26T22:20:00Z",
      "duration_ms": 120000
    }
  }
}
```

### Cache Key Computation

Each stage's cache key is a SHA-256 hash of:

```
SHA256(
    stage_name                   # e.g. "scene_detection"
    + "|" + video_hash           # SHA-256 of the video file
    + "|" + config_hash          # SHA-256 of relevant config params for this stage
    + "|" + version              # Cache format version (for migrations)
)
```

```python
import hashlib
import json
from pathlib import Path

def compute_video_hash(video_path: Path) -> str:
    """Compute SHA-256 of video file content (streaming for large files)."""
    h = hashlib.sha256()
    with open(video_path, "rb") as f:
        for chunk in iter(lambda: f.read(2**20), b""):  # 1 MB chunks
            h.update(chunk)
    return h.hexdigest()


def compute_config_hash(stage_name: str, config: dict) -> str:
    """Compute SHA-256 of the config parameters relevant to a stage.

    For each stage, we select only the config fields that affect its output.
    This means changing `ui_port` doesn't invalidate the transcribe cache.
    """
    relevant_params = STAGE_CONFIG_KEYS.get(stage_name, [])
    subset = {k: config.get(k) for k in relevant_params if k in config}
    normalized = json.dumps(subset, sort_keys=True, default=str)
    return hashlib.sha256(normalized.encode()).hexdigest()


def compute_stage_key(stage_name: str, video_hash: str, config_hash: str) -> str:
    """Compute the final cache key for a stage."""
    raw = f"{stage_name}|{video_hash}|{config_hash}|v1"
    return hashlib.sha256(raw.encode()).hexdigest()
```

### Cache Invalidation Strategies

| Strategy | Trigger | Mechanism |
|----------|---------|-----------|
| **Explicit** | `--force` CLI flag | Deletes manifest file entirely → full re-run |
| **Automatic (config)** | Config hash mismatch | Stage's config_hash in manifest != current config_hash |
| **Automatic (video)** | Video file hash mismatch | `video_hash` in manifest != current video_hash |
| **TTL-based** | Optional max-age per stage | `cached_at` timestamp + `max_age` → re-run if expired |
| **Dependency cascade** | Upstream stage re-ran | If stage A blob_hash changed, stage B (which reads A's output) must re-run |

**TTL config (optional):**
```python
# Per-stage TTL in seconds (None = no expiry)
STAGE_TTL = {
    "scene_detection": None,    # Scene boundaries don't change → cache forever
    "transcribe": 86400 * 7,   # Re-transcribe weekly (model may improve)
    "object_detection": None,  # YOLO results are deterministic
    "clip_classification": None,
}
```

### Stage Decorator Pattern

The cleanest integration pattern uses a decorator:

```python
import functools
from video_analysis.cache import PipelineCache

cache = PipelineCache(config)

@cache.cached_stage("scene_detection")
def _detect_scenes(self, video_path, video_id):
    # ... original implementation ...
    return scenes
```

**Decorator implementation (pseudo-code):**
```python
def cached_stage(self, stage_name: str):
    def decorator(func):
        @functools.wraps(func)
        def wrapper(instance, *args, **kwargs):
            video_path = kwargs.get("video_path") or args[0]
            video_hash = compute_video_hash(video_path)

            # If cache disabled, run function directly
            if not instance.config.pipeline_cache_enabled:
                return func(instance, *args, **kwargs)

            # Compute cache key
            config = instance.config.to_dict()  # or relevant subset
            config_hash = compute_config_hash(stage_name, config)
            stage_key = compute_stage_key(stage_name, video_hash, config_hash)

            # Check cache
            cached = self._load_stage(stage_name, stage_key, video_hash)
            if cached is not None:
                logger.info(f"Cache HIT for stage '{stage_name}'")
                return cached

            # Run and cache
            logger.info(f"Cache MISS for stage '{stage_name}' — executing...")
            result = func(instance, *args, **kwargs)
            self._save_stage(stage_name, stage_key, video_hash, config_hash, result)
            return result

        return wrapper
    return decorator
```

## Key Design Decisions

### Decision 1: Stage → blob key mapping (manifest) vs. direct path

**Option A (Recommended): Manifest-based.** A single `manifests/<video_hash>.json` tracks all stages for a video. This enables:
1. Quick check of what's cached without inspecting 20+ blob files
2. Atomic invalidation of entire video's cache (delete manifest)
3. Rich metadata (timestamps, durations, config hashes) for debugging
4. Easy "cache info" display for users

**Option B: Direct path.** Each stage has a deterministic path like `blobs/<stage_key>.pkl`. Simpler but loses metadata and makes inspection harder.

### Decision 2: Pickle vs. JSON serialization

**Pickle** for stage outputs since SceneInfo, FrameInfo, TranscriptSegment are dataclasses with nested objects. JSON serialization would require custom encoders/decoders. Security note: pickle in cache directory is a potential vector if an attacker gains write access — acceptable for a self-hosted, single-user application.

### Decision 3: What goes into the cache key hash

**Minimal approach (Recommended):** Only video hash + stage name + stage-relevant config params.

This means:
- Changing `ui_port` → no cache invalidations
- Changing `whisper_model` → only `transcribe` stage invalidates
- Changing `yolo_model` → only `object_detection` invalidates
- Changing `scene_threshold` → only `scene_detection` invalidates

**Maximal approach (not recommended):** Include all config params. Simple but causes spurious invalidations.

### Decision 4: Where to integrate the decorator

**Option A (Recommended): Decorator on private methods.** Each `_transcribe`, `_detect_scenes`, etc. gets a `@cache.cached_stage(...)` decorator. The caller `process()` remains unchanged — cache hits skip execution transparently.

**Option B: Wrap calls in `process()`.** Add `if cache_key in manifest: return manifest[cache_key]` checks before each stage call in `process()`. More explicit but duplicates logic.

**Option C: Abstract PipelineStage class.** Each stage becomes a `PipelineStage` subclass with a `run()` method. This is architecturally cleaner but requires significant refactoring of the existing 1673-line pipeline.py. **Defer to v0.25.0** as planned in the roadmap.

### Decision 5: Cache directory location

`data/cache/` (relative to `Config.data_dir`), matching the CHANGELOG v0.20.0 spec. Since `Config` resolves `data_dir` in `__post_init__`, the cache dir would be `Config.data_dir / "cache"`.

```python
# In Config.__post_init__:
self.cache_dir = self.data_dir / "cache"
self.cache_manifests_dir = self.cache_dir / "manifests"
self.cache_blobs_dir = self.cache_dir / "blobs"
```

```python
class PipelineCache:
    """Content-addressable SHA-256 cache for pipeline stages.

    Two-level structure:
        data/cache/manifests/<video_hash>.json  (stage → blob key mapping)
        data/cache/blobs/<sha256>.pkl            (serialized stage outputs)

    Cache invalidation:
        - Explicit:  delete manifest or call invalidate()
        - Automatic: config hash mismatch detected on lookup
        - TTL:       optional max-age per stage
    """

    def __init__(self, cache_dir: Path, enabled: bool = True):
        self._enabled = enabled
        self._cache_dir = Path(cache_dir)
        self._manifests_dir = self._cache_dir / "manifests"
        self._blobs_dir = self._cache_dir / "blobs"
        self._manifests_dir.mkdir(parents=True, exist_ok=True)
        self._blobs_dir.mkdir(parents=True, exist_ok=True)

    @staticmethod
    def compute_video_hash(video_path: Path) -> str:
        """Streaming SHA-256 of video file (handles multi-GB files)."""
        h = hashlib.sha256()
        with open(video_path, "rb") as f:
            for chunk in iter(lambda: f.read(1 << 20), b""):
                h.update(chunk)
        return h.hexdigest()

    @staticmethod
    def compute_config_hash(
        stage_name: str,
        config_dict: dict,
    ) -> str:
        """SHA-256 of stage-relevant config params only."""
        keys = STAGE_CONFIG_KEYS.get(stage_name, [])
        subset = {k: config_dict.get(k) for k in keys if k in config_dict}
        normalized = json.dumps(subset, sort_keys=True, default=str)
        return hashlib.sha256(normalized.encode()).hexdigest()

    @staticmethod
    def compute_stage_key(
        stage_name: str,
        video_hash: str,
        config_hash: str,
    ) -> str:
        """Final cache key for a stage execution."""
        raw = f"{stage_name}|{video_hash}|{config_hash}|v{CACHE_VERSION}"
        return hashlib.sha256(raw.encode()).hexdigest()

    # ── Load / Save ──────────────────────────────────────────────

    def _manifest_path(self, video_hash: str) -> Path:
        return self._manifests_dir / f"{video_hash}.json"

    def _blob_path(self, blob_hash: str) -> Path:
        return self._blobs_dir / f"{blob_hash}.pkl"

    def _load_manifest(self, video_hash: str) -> dict:
        path = self._manifest_path(video_hash)
        if path.exists():
            return json.loads(path.read_text())
        return {"version": CACHE_VERSION, "video_hash": video_hash, "stages": {}}

    def _save_manifest(self, video_hash: str, manifest: dict) -> None:
        path = self._manifest_path(video_hash)
        path.write_text(json.dumps(manifest, indent=2, default=str))

    def _load_stage(self, stage_key: str, blob_hash: str) -> Optional[Any]:
        """Load a cached stage output. Returns None if not found or corrupted."""
        path = self._blob_path(blob_hash)
        if not path.exists():
            return None
        try:
            with open(path, "rb") as f:
                return pickle.load(f)
        except Exception as e:
            logger.warning(f"Cache corrupt for blob {blob_hash}: {e}")
            path.unlink(missing_ok=True)
            return None

    def _save_stage(self, stage_key: str, result: Any) -> str:
        """Serialize and save a stage output. Returns the blob hash."""
        # Hash the serialized bytes so identical results deduplicate
        blob_bytes = pickle.dumps(result, protocol=5)
        blob_hash = hashlib.sha256(blob_bytes).hexdigest()
        path = self._blob_path(blob_hash)
        if not path.exists():
            path.write_bytes(blob_bytes)
        return blob_hash

    # ── Public API ───────────────────────────────────────────────

    def get_cached_stage(
        self,
        stage_name: str,
        video_hash: str,
        config_hash: str,
        max_age: Optional[float] = None,
    ) -> Optional[Any]:
        """Try to load a cached stage output.

        Args:
            stage_name:  e.g. "scene_detection"
            video_hash:  SHA-256 of video file
            config_hash: SHA-256 of stage-relevant config
            max_age:     Max cache age in seconds (None = no TTL check)

        Returns:
            Cached result, or None if cache miss.
        """
        if not self._enabled:
            return None

        manifest = self._load_manifest(video_hash)
        entry = manifest.get("stages", {}).get(stage_name)
        if entry is None:
            return None

        expected_key = self.compute_stage_key(stage_name, video_hash, config_hash)

        # Check config hash mismatch → automatic invalidation
        if entry.get("config_hash") != config_hash:
            logger.info(
                f"Cache invalidated for '{stage_name}': config changed "
                f"({entry.get('config_hash', '')[:8]} ≠ {config_hash[:8]})"
            )
            return None

        # Check cache key mismatch (e.g., version bump)
        if entry.get("stage_key") != expected_key:
            return None

        # Check TTL expiry
        if max_age is not None:
            cached_at = entry.get("cached_at", 0)
            if time.time() - cached_at > max_age:
                logger.info(f"Cache expired for '{stage_name}' (TTL={max_age}s)")
                return None

        # Load blob
        blob_hash = entry.get("blob_hash")
        if blob_hash is None:
            return None

        result = self._load_stage(expected_key, blob_hash)
        if result is not None:
            logger.info(f"Cache HIT [{stage_name}] ({entry.get('duration_ms', 0)}ms)")
            return result

        return None

    def set_cached_stage(
        self,
        stage_name: str,
        video_hash: str,
        config_hash: str,
        result: Any,
        duration_ms: Optional[float] = None,
    ) -> None:
        """Cache a stage output."""
        if not self._enabled:
            return

        stage_key = self.compute_stage_key(stage_name, video_hash, config_hash)
        blob_hash = self._save_stage(stage_key, result)

        manifest = self._load_manifest(video_hash)
        manifest.setdefault("stages", {})[stage_name] = {
            "stage_key": stage_key,
            "config_hash": config_hash,
            "blob_hash": blob_hash,
            "cached_at": time.time(),
            "duration_ms": duration_ms or 0,
        }
        self._save_manifest(video_hash, manifest)
        logger.info(
            f"Cached [{stage_name}] key={stage_key[:12]}… "
            f"blob={blob_hash[:12]}…"
        )

    def invalidate(
        self,
        video_hash: str,
        stage_name: Optional[str] = None,
    ) -> None:
        """Invalidate cache for a video.

        Args:
            video_hash: SHA-256 of video file.
            stage_name: If set, invalidate only this stage. If None, invalidate all.
        """
        manifest = self._load_manifest(video_hash)
        if not manifest.get("stages"):
            return

        if stage_name:
            manifest["stages"].pop(stage_name, None)
            logger.info(f"Invalidated cache for '{stage_name}' ({video_hash[:8]}…)")
        else:
            manifest["stages"] = {}
            logger.info(f"Invalidated ALL cache ({video_hash[:8]}…)")

        self._save_manifest(video_hash, manifest)

    # ── Decorator ────────────────────────────────────────────────

    def cached_stage(
        self,
        stage_name: str,
        max_age: Optional[float] = None,
    ):
        """Decorator that caches a pipeline stage method.

        The decorated method must have ``video_path`` as its first argument
        (or ``video_path`` as a keyword argument).

        Usage::

            class VideoPipeline:
                def __init__(self, config):
                    self._cache = PipelineCache(config.cache_dir,
                                                config.pipeline_cache_enabled)

                @self._cache.cached_stage("scene_detection")
                def _detect_scenes(self, video_path, video_id):
                    ...
        """
        import functools

        def decorator(func: Callable):
            @functools.wraps(func)
            def wrapper(instance, *args, **kwargs):
                # Extract video_path from args/kwargs
                if args:
                    video_path = args[0]
                else:
                    video_path = kwargs.get("video_path")
                if video_path is None:
                    return func(instance, *args, **kwargs)

                video_path = Path(video_path)
                video_hash = self.compute_video_hash(video_path)

                # Compute config hash from instance.config
                config_dict = vars(instance.config) if hasattr(instance, "config") else {}
                config_hash = self.compute_config_hash(stage_name, config_dict)

                # Try cache
                cached = self.get_cached_stage(
                    stage_name, video_hash, config_hash, max_age=max_age
                )
                if cached is not None:
                    return cached

                # Execute and cache
                start = time.time()
                result = func(instance, *args, **kwargs)
                duration_ms = (time.time() - start) * 1000

                self.set_cached_stage(
                    stage_name, video_hash, config_hash, result,
                    duration_ms=duration_ms,
                )
                return result

            return wrapper

        return decorator
```

## Upstream Dependency Tracking

Since the pipeline is sequential (chain DAG), when an upstream stage is re-executed, all downstream stages that depend on its output must also re-run. This is tracked by:

1. **After each stage executes**: record the blob_hash in the manifest
2. **Before each stage runs**: check if *any* of its direct upstream dependencies have different blob_hashes than recorded
3. **If upstream changed**: force re-run even if this stage's own config_hash matches

For the current linear pipeline, this is simple:
```
If stage N was cached but stage N-1 just re-ran → stage N must re-run too
```

This is implemented by storing a "manifest version" or simply checking: after a miss on stage N-1, the manifest entry for stage N is automatically invalidated.
