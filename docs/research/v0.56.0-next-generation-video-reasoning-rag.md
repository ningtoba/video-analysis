1. **GRPO/RL-based post-training** for video MLLMs (the "Video-R1" era)
2. **Structured RAG** — Event-Causal graphs, spatio-temporal graphs, intent-aware retrieval
3. **Streaming thinking** — "Watch and think simultaneously" with amortized reasoning
4. **Competence-aware reasoning** — adaptive compute allocation via RL rewards
5. **Multi-hop evidence retrieval** — the primary bottleneck in long-video QA

## 1. Video Reasoning Post-Training (GRPO/RL Era)

### 1.1 VideoKR: Knowledge- and Reasoning-Intensive Video Understanding
- **Paper:** arXiv:2606.05259 (ICML 2026 Spotlight)
- **Key insight:** First large-scale training corpus (315K examples over 145K expert-domain videos)
  designed for knowledge-intensive video reasoning. Uses human-in-the-loop, skill-oriented
  generation pipeline. Standard SFT → GRPO pipeline outperforms prior post-training approaches.
- **Relevance:** Could fine-tune InternVideo3-8B or Qwen3-VL on video reasoning tasks.
- **Data:** 315K video reasoning examples, 145K CC-licensed videos, CoT rationales.
- **RTX 4070 feasibility:** GRPO on 8B models requires ~24GB. Not viable directly.
  But could use the **training data** to do SFT + LoRA on InternVideo3-8B via Unsloth/QLoRA.

### 1.2 DynFrame: Learnable Frame Sampling via GRPO
- **Paper:** arXiv:2605.26680
- **Key insight:** Emits temporal window AND sampling density as native tokens in a single
  autoregressive pass. Introduces **Segment-Decoupled GRPO (SD-GRPO)** — separate RL
  rewards for "where to look" tokens vs "how to answer" tokens.
- **DynFrame-4B:** Competitive with 7B-8B baselines. **DynFrame-8B:** SOTA on most metrics.
- **Relevance:** Could implement a lightweight version in the existing frame_compression.py
  as a learnable frame selector driven by the LLM Provider.

### 1.3 CARE: Competence-Aware Reward Shaping
- **Paper:** arXiv:2606.19927
- **Key insight:** EMA of pass rates → progressive training stages that shift reward
  preference from exploration (long reasoning) to efficiency (concise reasoning).
  Inverted-U reasoning length trajectory. No inference-time overhead.
- **Relevance:** Techniques applicable to RL-based fine-tuning pipelines.

### 1.4 Robust-TO: Per-Frame Trustworthiness (v2)
- **Paper:** arXiv:2606.26904
- **Key insight:** Under realistic perturbations (motion blur, glare, occlusion), frontier
  video reasoning models drop 15-30% accuracy. Robust-TO adds per-frame trust scoring
  with confidence-cost GRPO reward. Achieves 56.4% avg accuracy (+10.6% over open-source,
  beating Gemini-2.5-Pro).
- **Relevance:** The project already has `agent_confidence.py` (v0.50.0). This could
  be upgraded with the Robust-TO v2 confidence-cost GRPO mechanism.

## 2. Structured RAG for Long Videos

### 2.1 Event-Causal RAG
- **Paper:** arXiv:2605.06185
- **Key insight:** Segments streaming videos into **semantically coherent events** (not fixed
  clips), represents each event as a **State-Event-State (SES) graph** capturing state
  transitions. Merges into a global Event Knowledge Graph with dual-store memory
  (semantic matching + causal-topological retrieval). Bidirectional retrieval strategy.
- **Relevance:** **HIGH.** The project already has:
  - `knowledge_graph.py` — SQLite entity/relationship store
  - `scene_graph.py` — VGent-style graph generation with K-hop expansion
  - `streaming.py` — chunked/live streaming analysis
  
  **Gap:** No event segmentation. No SES graph representation. No causal-topological retrieval.
  This could be a **major v0.56.0 feature** — Event-Causal RAG module integrating
  with the existing KnowledgeGraph and scene graph infrastructure.

### 2.2 VideoStir: Spatio-Temporally Structured Intent-Aware RAG
- **Paper:** arXiv:2604.05418 (ACL 2026)
- **Key insight:** Structures video as spatio-temporal graph at clip level, then performs
  **multi-hop retrieval** across distant yet contextually related events. Introduces
  MLLM-backed **intent-relevance scorer** that retrieves frames based on alignment with
  the query's reasoning intent. Curates IR-600K dataset for frame-query intent alignment.
- **Relevance:** The project already has spatio-temporal scene graphs (scene_graph.py)
  and multi-hop retrieval (rag.py). VideoStir's **intent-relevance scoring** could be
  integrated as a new retrieval stage.

### 2.3 EgoGraph: Temporal Knowledge Graph for Egocentric Video
- **Paper:** arXiv:2602.23709
- **Key insight:** Training-free dynamic KG construction for ultra-long videos (multi-day).
  Egocentric schema unifying persons, objects, locations, events with cross-entity
  temporal dependencies. SOTA on EgoLifeQA and EgoR1-bench.
- **Relevance:** The project's `knowledge_graph.py` already tracks entities/timelines.
  EgoGraph's **dynamic temporal dependency modeling** could enhance the KG module.

### 2.4 VGent: Graph-based Retrieval-Reasoning-Augmented Generation
- **Paper:** arXiv:2510.14032 (NeurIPS 2025 Spotlight)
- **Key insight:** Represents videos as structured graphs with semantic relationships
  across clips. Intermediate reasoning step with structured verification to reduce
  retrieval noise. +3.0-5.4% over base models on MLVU, +8.6% over prior video RAG.
- **Relevance:** Already partially implemented in scene_graph.py. Could be deepened
  with structured verification (in rag.py or as a new module).

## 3. Streaming Video Reasoning

### 3.1 Video Streaming Thinking (VST)
- **Paper:** arXiv:2603.12262
- **Key insight:** "Watch and think simultaneously" — activates reasoning over incoming
  video clips during streaming, amortizing LLM reasoning latency over video playback.
  VST-SFT adapts offline VideoLLM to causal streaming reasoning. VST-RL provides
  end-to-end improvement via multi-turn video interaction.
  VST-7B: 79.5% on StreamingBench, 15.7× faster than Video-R1, +5.4% on VideoHolmes.
- **Relevance:** **HIGH.** The project already has streaming.py with chunked/live
  stream analysis. VST's **streaming thinking** paradigm could be integrated into
  StreamingPipeline — reasoning over chunks as they arrive.

### 3.2 Fre-Res: Frequency-Residual Video Token Compression
- **Paper:** arXiv:2605.16366
- **Key insight:** Separates spatial fidelity (sparse high-fidelity anchors) from temporal
  evolution (compact residual-frequency tokens). Uses temporal 1D-DCT on inter-frame
  residual trajectories in vision-latent space. Achieves favorable accuracy-efficiency
  trade-off.
- **Relevance:** The project has DINOv2 frame compression (frame_compression.py).
  Fre-Res offers a **frequency-domain alternative** that's more compatible with
  video MLLM token spaces.

### 3.3 OneClip-RAG: One-shot Clip Retrieval-Augmented Generation
- **Paper:** arXiv:2512.08410
- **Key insight:** Uses video clips for augmented understanding (knowledge integrity +
  semantic coherence). Novel **query-guided video chunking** that unifies clip chunking
  and cross-modal retrieval in one step. Boosts Qwen3-VL 8B to GPT-5 level on MLVU.
  Enables LLaVA-Video to understand 1-hour videos in <1.2 min on single 4090.
- **Relevance:** Could replace the existing chunking strategy in StreamingPipeline
  with query-guided chunking.

## 4. Multi-Hop Evidence Retrieval

### 4.1 LongVidSearch Benchmark
- **Paper:** arXiv:2603.14468
- **Key insight:** First benchmark enforcing strict multi-hop retrieval necessity.
  Hop-k questions require exactly k evidence clips. Even GPT-5 achieves only 42.43%
  accuracy. **Retrieval planning is the primary bottleneck** in long-video QA.
- **Relevance:** Validates the importance of the project's existing multi-hop retrieval
  in rag.py. Suggests enhancing the agent's retrieval planning (agent.py) with
  standardized tool interfaces.

### 4.2 UpstreamQA: Explicit Reasoning Modules
- **Paper:** arXiv:2604.23145
- **Key insight:** Disentangles video reasoning into explicit upstream modules
  (object identification, scene context generation) before downstream VideoQA.
  Using LRMs (o4-mini, Gemini 2.5 Pro) for explicit reasoning traces.
- **Relevance:** The project's `orchestra.py` (multi-agent orchestrator) already does
  something similar. UpstreamQA validates the modular reasoning approach.

## 5. Scene Graph Generation Advances

### 5.1 SceneGraphVLM: Compact VLM-based Scene Graph Generation
- **Paper:** arXiv:2605.13667
- **Key insight:** Uses small VLMs with token-efficient TOON format serialization.
  Two-stage training: SFT → RL with hallucination-aware rewards. ~1-second latency.
  Conditions each frame on previous graph for lightweight temporal context.
- **Relevance:** The project's scene_graph.py generates graphs via Hermes/LLM provider.
  SceneGraphVLM's RL-based training with hallucination rewards could inspire
  a higher-quality scene graph implementation.

### 5.2 Synthetic Visual Genome 2: Large-Scale Spatio-Temporal Scene Graphs
- **Paper:** arXiv:2602.23543
- **Key insight:** Extracts large-scale spatio-temporal scene graphs from videos
  algorithmically. Provides training data for VidSGG models.
- **Relevance:** Dataset source for improving the scene graph module.

## 6. Fine-Tuning & Adapter Approaches

### 6.1 APT-Tune: Parameter-Efficient Fine-Tuning for Video Understanding
- **Paper:** arXiv:2606.18586
- **Key insight:** Atomic Physical Transitions (APTs) — minimal temporally localized
  state changes as causal supervision. Only 11M LoRA parameters on Qwen3-VL-2B.
  Improves APT recall while preserving event-level video understanding.
- **Relevance:** **HIGH.** With only 11M LoRA params, this is viable on RTX 4070 12GB.
  APT supervision could be used to fine-tune existing backends for causal
  physical understanding.

### 6.2 VCIFBench DPO Dataset
- **Paper:** arXiv:2606.04588
- **Key insight:** Complex Instruction Following for video understanding. 306 test
  instructions, 540-pair DPO preference dataset, 30 conflict diagnostics.
  DPO training on VCIFBench data improves instruction-following.
- **Relevance:** DPO on 540 pairs is feasible. Could fine-tune SmolVLM2 (2.2B)
  which fits in ~5GB VRAM with LoRA.

## 7. Recommended v0.56.0 Direction

Based on synthesis of all findings, the highest-impact feature for v0.56.0 is:

### Event-Causal RAG with Streaming Thinking

**Rationale:**
1. The project already has: knowledge graph, scene graph, streaming pipeline, RAG
2. No existing module implements **event-level segmentation** or **causal reasoning**
3. Event-Causal RAG (arXiv:2605.06185) directly addresses the #1 bottleneck:
   **retrieval planning** instead of clip-based flat retrieval
4. VST (arXiv:2603.12262) streaming thinking integrates naturally with the
   existing StreamingPipeline
5. Both are **training-free** approaches — no GPU needed for fine-tuning,
   they use existing MLLM + LLM backends

**Implementation plan:**
1. Add `video_analysis/event_rag.py` — Event-Causal RAG module:
   - Event segmentation (semantic boundaries, not fixed clips)
   - State-Event-State (SES) graph construction
   - Dual-store memory (semantic + causal-topological)
   - Bidirectional retrieval strategy
2. Add `video_analysis/streaming_think.py` — Streaming Thinking:
   - "Watch and think simultaneously" in StreamingPipeline
   - Amortized reasoning over incoming chunks
   - Multi-turn streaming interaction
3. Add config fields and env vars for both features
4. Wire into existing RAG (rag.py) and streaming (streaming.py) pipelines
5. Add Gradio tab for Event-Causal KG exploration
6. Add REST API endpoints
7. 25+ tests

**Alternative (if above is too large):** DynFrame-style learnable frame sampling
via SD-GRPO in frame_compression.py + streaming pipeline — lighter but still impactful.
