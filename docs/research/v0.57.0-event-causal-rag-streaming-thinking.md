## What Was Built

### 1. Event-Causal RAG (`video_analysis/event_rag.py`)

**Core insight:** The project's existing RAG operates at the *clip/scene* level,
but real-world video understanding requires *event-level* reasoning — grouping
multiple scenes into semantically coherent narrative units.

**Components:**

| Component | Description | Paper |
|-----------|-------------|-------|
| `Event` | Data class for semantically coherent video events | — |
| `EventSegmenter` | 3-tier segmentation: LLM → transcript-coherence → temporal-grid | arXiv:2605.06185 |
| `SESGraph` | State-Event-State graph with causal + temporal edges | arXiv:2605.06185 |
| `SemanticStore` | Dense embedding retrieval for event descriptions (ChromaDB/fallback) | — |
| `CausalTopologicalStore` | Graph BFS for forward (prediction) and backward (explanation) retrieval | arXiv:2605.06185 |
| `DualStoreMemory` | Fused retrieval combining semantic + causal-topological scores | arXiv:2605.06185 |
| `EventCausalRAG` | Orchestrator wrapping all components | — |

**Architecture:**
```
EventSegmenter → SESGraph → DualStoreMemory → BidirectionalRetriever
                                  │
                    ┌─────────────┴─────────────┐
                    ▼                           ▼
              SemanticStore             CausalTopologicalStore
              (ChromaDB/BGE-VL)          (Graph BFS)
```

**Key design decisions:**
- 3-tier segmentation strategy ensures graceful degradation (LLM → transcript → temporal)
- DualStoreMemory fuses semantic + causal scores with configurable weights
- CausalTopologicalStore supports K-hop BFS with path descriptions
- All components are training-free — use existing MLLM/LLM backends

### 2. Streaming Thinking (`video_analysis/streaming_think.py`)

**Core insight:** The project's existing `StreamingPipeline` processes chunks
passively. Streaming Thinking adds *active reasoning* — "watch and think
simultaneously" — amortizing LLM reasoning latency over video playback.

**Components:**

| Component | Description | Paper |
|-----------|-------------|-------|
| `ThoughtState` | Accumulated reasoning across chunks | arXiv:2603.12262 |
| `StreamingThought` | Per-chunk thinking step with insights & predictions | — |
| `StreamingThinkingPipeline` | Wraps StreamingPipeline with thinking layer | arXiv:2603.12262 |

**Per-chunk thinking process:**
1. Entity accumulation (frequency tracking across chunks)
2. Insight generation (object, scene, transcript, trend analysis)
3. **Forward thinking** (causal prediction) — what might happen next?
4. **Backward thinking** (causal explanation) — how does this relate to earlier content?
5. Question generation — what knowledge gaps exist?
6. State summary update with truncated transcript history

### 3. Configuration

New config fields in `video_analysis/config.py`:

| Variable | Default | Description |
|----------|---------|-------------|
| `EVENT_CAUSAL_RAG_ENABLED` | `false` | Enable Event-Causal RAG module |
| `EVENT_SEGMENTATION_STRATEGY` | `auto` | Segmentation: auto, llm, transcript, temporal |
| `EVENT_CAUSAL_TOP_K` | `10` | Max events from bidirectional retrieval |
| `EVENT_CAUSAL_SEMANTIC_WEIGHT` | `0.5` | Semantic vs causal store weight |
| `EVENT_MAX_DURATION_SECONDS` | `300` | Max event duration |
| `STREAMING_THINKING_ENABLED` | `false` | Enable Streaming Thinking |
| `STREAMING_THINKING_INTERVAL` | `1` | Think on every Nth chunk |
