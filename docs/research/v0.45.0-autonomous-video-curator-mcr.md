# v0.45.0 — Autonomous Video Curator: MCR Closed-Loop Exploration

## Inspiration

This feature draws from two cutting-edge 2026 video AI research directions:

### InternVideo3: Multimodal Contextual Reasoning (arXiv:2606.12195, Jun 2026)

InternVideo3 introduces **MCR (Multimodal Contextual Reasoning)** — a framework
that treats video understanding as a **closed-loop process over a shared,
evolving context** containing observations, instructions, reasoning, tool actions,
and memory. This frames long-video understanding as evidence accumulation and
verification, rather than one-shot analysis.

Key innovations from InternVideo3 that inspired the curator:
- **Closed-loop MCR**: Observation → Analysis → Memory → Reasoning → Action
- **M²LA (Multimodal Multi-head Latent Attention)**: Token-preserving KV-cache
  compression for efficient long-context processing
- **Staged training**: Continued pretraining → short-to-long SFT → RL → distillation
- **Evidence-grounded retrieval**: Video agent uses retrieval tools for grounded behavior

### HKUDS VideoAgent: All-in-One Agentic Framework

VideoAgent provides a comprehensive framework for video understanding, editing,
and remaking through:
- **Intent analysis & autonomous planning**
- **Multi-modal tool use architecture**
- **Seamless natural language interaction**

## Design Decisions

### Why a Curator, not another Agent?

The existing `VideoUnderstandingAgent` is **reactive** — it answers questions
about videos that have already been processed. The `VideoCurator` is
**proactive** — it initiates exploration autonomously.

This fills a gap: users often don't know what to ask about a video. The curator
discovers what's interesting *for* them.

### Heuristic vs Learned Curiosity

The `CuriosityStrategy` uses **heuristic rules** rather than an LLM call for
every decision. This keeps the system:
1. **Fast**: No round-trip to an LLM for action selection
2. **Deterministic**: Reproducible exploration behavior
3. **Lightweight**: Runs even without an LLM available

The 6 strategy rules form a priority chain:
1. No observations → broad sweep
2. Unanswered questions → search topic
3. Temporal gaps → sample later portions
4. Knowledge gaps → investigate gaps
5. High curiosity → deep focus on rich entities
6. Default → generate curiosity questions

### Knowledge Persistence

The curator saves its knowledge state as JSON, enabling cross-session curation.
This means a user can:
1. Run short curation on a 2-hour video
2. Come back later and resume from where the curator left off
3. Layer new findings on top of previous observations

### Saturation Detection

The curator detects when it's no longer discovering new information and stops
early. This prevents:
- Infinite loops
- Wasting iterations on already-explored content
- Generating repetitive reports

## Architecture

```
┌─────────────────────────────────────────────────────────────┐
│                  MCR Closed Loop                            │
│                                                             │
│  OBSERVE ──→ ANALYZE ──→ MEMORIZE ──→ REASON ──→ ACT      │
│    ▲                                                       │
│    └─────────────────────── Loop ──────────────────────────┘
│                                                             │
│  Tools:                                                     │
│  ├── analyze_frames (Video MLLM)                            │
│  ├── search_rag (ChromaDB retrieval)                        │
│  ├── detect_objects (YOLO)                                  │
│  ├── extract_text (PaddleOCR)                               │
│  └── search_transcript (Whisper transcript)                 │
│                                                             │
│  Outputs:                                                   │
│  ├── Markdown Report (to_markdown())                        │
│  ├── JSON Report (to_json())                                │
│  └── Knowledge State (data/curations/*_knowledge.json)      │
└─────────────────────────────────────────────────────────────┘
```

## Data Flow

1. **Phase 1 — Broad Sweep**: Sample 5 strategic points (early, early-mid, mid,
   mid-late, near-end) + scene boundaries → analyze with Video MLLM

2. **Phase 2 — MCR Loop** (up to `max_iterations`):
   - REASON: `CuriosityStrategy.suggest_next_action()` selects next action
   - ACT: Execute the action (analyze_frames, search_rag, detect_objects, etc.)
   - OBSERVE: Collect findings as `CuratorObservation`
   - MEMORIZE: Update `CuratorKnowledge` with new entities, observations, gaps

3. **Phase 3 — Report Generation**:
   - Cluster observations by temporal proximity
   - Extract entity summary by type
   - Build timeline of key moments
   - Render Markdown and JSON reports

## Future Directions

1. **LLM-powered curiosity**: Replace heuristic strategy with LLM-based action
   selection for more nuanced exploration decisions

2. **Cross-video knowledge**: Link entities across multiple curated videos
   (e.g., "Jane Doe appears in videos A and C")

3. **Active learning**: Use confidence scores to identify uncertain observations
   and request human verification

4. **Hierarchical exploration**: Multi-scale exploration (coarse → fine-grained
   focus on interesting regions)

5. **Streaming curator**: Apply MCR loop to live streams for real-time
   autonomous analysis
