## 0.39.0 (2026-06-27) — Self-Contained LLM Provider

### 🧠 Self-Contained LLM Provider (LLMProvider Abstraction)

The platform no longer has a hard dependency on Hermes CLI (`hermes chat -q`) for
all LLM calls. A new `LLMProvider` abstraction layer (`video_analysis/llm_provider.py`)
provides a unified interface for LLM backends:

- **`HermesProvider`** (default) — existing `hermes chat -q` subprocess, fully
  backward-compatible
- **`OpenAIProvider`** — any OpenAI-compatible API endpoint (vLLM, Ollama,
  llama.cpp, TGI, OpenAI API, Azure OpenAI)
- **`auto` mode** — tries OpenAI-compatible API first, falls back to Hermes CLI

This abstraction replaces all 5 direct `hermes chat -q` subprocess calls across
the codebase:

| Module | Previous | Now |
|--------|----------|-----|
| `chat.py` | `hermes chat -q` subprocess | `llm.chat()` via LLMProvider |
| `self_check.py` | 2× `hermes chat -q` subprocess calls | `llm.structured_chat()` + `llm.chat()` via LLMProvider |
| `query_router.py` | `hermes chat -q` subprocess | `llm.structured_chat()` via LLMProvider |
| `chapters.py` | `hermes chat -q` subprocess | `llm.structured_chat()` via LLMProvider |

### 🔧 Configuration

| Variable | Default | Description |
|----------|---------|-------------|
| `LLM_PROVIDER` | `hermes` | LLM backend (`hermes`, `openai`, `auto`) |
| `OPENAI_API_BASE` | `http://localhost:11434/v1` | OpenAI-compatible API URL |
| `OPENAI_API_KEY` | (empty) | API key (can be empty for local servers) |
| `OPENAI_MODEL` | `qwen2.5` | Model name for the API |
| `OPENAI_MAX_TOKENS` | `2048` | Max tokens for API calls |
| `LLM_TIMEOUT` | `120` | Timeout in seconds |
| `LLM_TEMPERATURE` | `0.3` | LLM temperature |

### 💡 Structured Output

Both providers support `structured_chat()` which returns parsed JSON:
- Handles direct JSON, markdown code blocks (```json ... ```), and
  embedded JSON extraction
- Enables reliable JSON schema-constrained responses from any LLM backend

### 🧪 Tests

- **38 new tests** (`tests/test_llm_provider.py`) — all pass in <0.3s
- Covers: Config (3 tests), HermesProvider chat/success/failure/timeout (9 tests),
  HermesProvider structured output with JSON parsing (3 tests),
  OpenAIProvider chat/success/failure/system/available (8 tests),
  OpenAIProvider structured output (2 tests),
  Provider factory caching/fallback/force/unknown (9 tests),
  URL formatting (2 tests)
- All HermesProvider tests mock `subprocess.run` — no Hermes CLI needed
- All OpenAIProvider tests mock `requests` — no API endpoint needed

### 🔧 Integration

- All 5 existing modules now accept an optional `llm=` parameter for dependency
  injection (useful for testing with mock providers)
- Each module lazily initializes its own LLM provider from env vars when none
  is provided
- Provider instances are cached per type to reuse connections/sessions
