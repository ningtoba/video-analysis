## Overview

v0.31.0 introduces a standalone **ColBERT-Att** re-ranker (Patel & Dutta,
arXiv:2603.25248, Mar 2026) that enhances the existing late-interaction
retrieval by integrating BERT attention weights into the MaxSim scoring
function, providing a +1-3% recall improvement with zero additional training.

---

## 1. ColBERT-Att: Attention-Weighted Late Interaction

### 1.1 Motivation

Standard ColBERTv2 computes relevance as:

```
score(Q, D) = Σ_{q_i ∈ Q} max_{d_j ∈ D} (E_{q_i} · E_{d_j})
```

This MaxSim operation treats **all** query and document token matches as equally
important — but intuitively, some tokens carry more semantic weight than others.
In the query "Who is going to study?", the words "going" and "to" have high
cosine similarity with documents containing those words, but they're semantically
unimportant. Conversely, "study" carries high semantic weight but may have low
cosine similarity with related words like "school" or "learning".

### 1.2 ColBERT-Att Scoring

ColBERT-Att addresses this by incorporating BERT's last-layer attention weights
into the scoring function:

```
score(Q, D) = Σ_{q_i ∈ Q} α_i · max_{d_j ∈ D} (β_j · cos_sim(E_q_i, E_d_j))
```

Where:
- **E_q_i, E_d_j** — ColBERTv2 token embeddings (128-dim projected BERT hidden states)
- **α_i** — normalised attention weight for query token i (how much other query tokens attend to it)
- **β_j** — normalised attention weight for document token j (how much other doc tokens attend to it)
- **cos_sim** — cosine similarity between normalised token embeddings

### 1.3 Attention Extraction Method

The attention weights are extracted from BERT's last transformer layer:

1. **Forward pass** through ColBERTv2 with `output_attentions=True`
2. **Last-layer attention tensor** shape: `(B, num_heads, T, T)`
3. **Average over heads** → `(B, T, T)` — the average attention from any token to any other
4. **Sum over source dimension** (column-wise) → token *importance*: how much attention each token *receives* from all other tokens
5. **Softmax normalisation** over non-padding tokens → α_i (sums to 1 per sequence)
6. **Same process** for query (single pass) and each document (batched)

### 1.4 Implementation

The implementation is a **standalone module** (`video_analysis/colbert_att_reranker.py`)
that:

- Loads `colbert-ir/colbertv2.0` via HuggingFace `transformers` (uses `ColBertModel`
  / `HF_ColBERT` from transformers 5.12+ if available, with `AutoModel` fallback)
- Sets `output_attentions=True` to get BERT attention weights as a forward-pass side-effect
- Extracts both token embeddings (`last_hidden_state`) and attention weights
- Computes attention-weighted MaxSim per query-document pair
- Supports `rerank(query, documents, top_k)` interface matching the existing
  `ColBERTReranker` API
- Provides `unload()` to free ~2GB VRAM after re-ranking

### 1.5 Comparison to Existing ColBERTv2

| Aspect | ColBERTv2 (existing) | ColBERT-Att (new) |
|--------|---------------------|-------------------|
| Scoring | `Σ max_sim(E_q, E_d)` | `Σ α · max(β · cos_sim(E_q, E_d))` |
| Attention | Ignored | BERT last-layer weights |
| Accuracy | Baseline | +1-3% recall (MS-MARCO, BEIR, LoTTE) |
| Latency | Fastest | +~5% (attention extraction is cheap) |
| Training | Pre-trained | Frozen checkpoint, no fine-tuning |
| Deps | ragatouille | transformers (already required) |
| VRAM | ~2-3 GB | ~2 GB |

### 1.6 Why Not Use RAGatouille?

The existing `ColBERTReranker` wraps RAGatouille, which abstracts away the model
internals and does not expose attention weights. The new `ColBERTAttReranker`
loads ColBERTv2 directly via `transformers` (`ColBertModel` / `HF_ColBERT`),
giving us access to the full BERT encoder output including attention tensors.
This means:
- No RAGatouille dependency needed
- Full access to model internals
- Lighter footprint (~2 GB vs ~2-3 GB with RAGatouille's overhead)

## 2. Integration

### 2.1 Config

| Env Var | Default | Description |
|---------|---------|-------------|
| `COLBERT_ATT_RERANKER_ENABLED` | `false` | Enable ColBERT-Att re-ranking |

### 2.2 Pipeline Integration

ColBERT-Att plugs into `VideoRAG.retrieve()` as a final re-ranking stage,
running after:
1. ChromaDB initial retrieval
2. Cross-encoder re-ranking (`_rerank()`)
3. Optional ColBERTv2 re-ranking (`_rerank_colbert()`)
4. **ColBERT-Att re-ranking** (`_rerank_colbert_att()`) — new

The model is loaded lazily on first use and unloaded after re-ranking to free
VRAM for subsequent pipeline stages.

### 2.3 Cache Integration

The `colbert_att_reranker_enabled` config field is tracked in the pipeline
cache (cache.py) to invalidate cached RAG indexing results when the re-ranker
configuration changes.
