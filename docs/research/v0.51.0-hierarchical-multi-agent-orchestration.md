# v0.51.0 Deep-Dive Research — Hierarchical Multi-Agent Video Reasoning Orchestrator

## Executive Summary

Version 0.51.0 introduces a **Hierarchical Multi-Agent Video Reasoning Orchestrator** inspired by two cutting-edge 2026 research directions:

1. **HiCrew** (arXiv:2604.21444, April 2026) — Hierarchical multi-agent reasoning for long-form video understanding using a Hybrid Tree structure that preserves temporal topology while performing relevance-guided hierarchical clustering, a Question-Aware Captioning mechanism, and a Planning Layer that dynamically orchestrates agent collaboration based on question complexity.

2. **Orchestra-o1** (arXiv:2606.13707, June 2026) — Omnimodal agent orchestration framework achieving SOTA on OmniGAIA by enabling modality-aware task decomposition, online sub-agent specialization, and parallel sub-task execution via DA-GRPO reinforcement learning.

The new `video_analysis/orchestra.py` module provides a complete hierarchical multi-agent architecture that sits **above** the existing `VideoUnderstandingAgent`, transforming it from a flat rule-based tool dispatcher into a dynamic, LLM-powered multi-agent system.

## Key Components

### 1. Hybrid Tree Structure

After a video is processed by the pipeline, scene information from the `VideoIndex` is organized into a hierarchical tree:

```
Video (root)
├── Act 1 / Opening (high-level semantic cluster)
│   ├── Scene 1 (fine-grained shot boundary)
│   ├── Scene 2
│   └── Scene 3
├── Act 2 / Main Content
│   ├── Scene 4
│   ├── Scene 5
│   └── Scene 6
└── Act 3 / Closing
    ├── Scene 7
    └── Scene 8
```

The clustering uses transcript content similarity (via sentence-transformers) combined with temporal proximity — shots close in time AND semantically similar are grouped together.

### 2. Question-Aware Router Agent

Instead of the current `VideoUnderstandingAgent.query()` which uses hard-coded keyword matching to dispatch tools, the new **RouterAgent** (powered by LLMProvider):

- Analyzes the question's modality needs: visual / textual / temporal / action / entity / multi-hop
- Generates a structured `RoutePlan` with:
  - Sub-questions to delegate to specialist agents
  - Tool dependencies (what must run before what)
  - Confidence thresholds for early stopping
- Dynamically selects which specialist agents to activate based on the question

### 3. Specialist Sub-Agents (Parallel Execution)

| Agent | Tool | Description |
|-------|------|-------------|
| **VisualAnalyst** | analyze_frames | Video MLLM frame analysis with intent-driven prompts |
| **RAGSearcher** | search_rag | Vector search with query refinement |
| **TranscriptAnalyst** | search_transcript | Transcript search + temporal grounding |
| **ObjectDetectorAgent** | detect_objects | YOLO detection + entity tracking |
| **OCRAgent** | extract_text | OCR text extraction |
| **ConfidenceAuditor** | (cross-validation) | Cross-validate using v0.50.0 agent_confidence |
| **SummarizerAgent** | summarize_video | Structured video summarization |

### 4. Evidence Synthesis with Confidence

The **EvidenceSynthesizer** uses:
- `EvidenceTrustScorer` from v0.50.0 `agent_confidence.py` for per-evidence confidence
- `EvidenceWeighter` from v0.50.0 for tiered weighting (high/medium/low)
- Source attribution with timestamp citations
- Weighted combination across all agent results

### 5. Parallel Execution Strategy

- **Independent agents** (RAGSearcher, VisualAnalyst) run via `ThreadPoolExecutor`
- **Dependent chains** (ObjectDetector → VisualAnalyst ← ConfidenceAuditor) respect ordering
- **Early stopping**: if aggregated confidence exceeds threshold, remaining agents skip

## Config Fields

| Variable | Default | Description |
|----------|---------|-------------|
| `ORCHESTRA_ENABLED` | `false` | Enable hierarchical multi-agent orchestration |
| `ORCHESTRA_MAX_AGENTS` | `5` | Max specialist agents per query |
| `ORCHESTRA_CONFIDENCE_THRESHOLD` | `0.5` | Confidence threshold for early stopping |
