## Executive Summary

This research update found **several genuinely new models and methods** not covered in the previous v0.17.0 research. The most significant discoveries:

1. **Keye-VL-2.0-30B-A3B** (Kwai, June 2026) -- MoE model that activates only **3B parameters** for inference, achieving SOTA on TimeLens, Video-MME-v2, and LongVideoBench. First adaptation of DeepSeek Sparse Attention to multimodal architectures.
2. **Marlin-2B** (NemoStation, May 2026) -- 2B video MLLM, 15.3k downloads, extremely popular. Built on Qwen2.5-VL-2B base.
3. **EgoReasoner** (March 2026) -- 3B egocentric 4D reasoning model, surpasses Qwen2.5-VL-7B by 10+ points on HD-EPIC.
4. **NVIDIA Cosmos-Embed1** (May 2025/2026) -- Joint video-text embedder (1B params), SOTA on AV and robotics retrieval.
5. **V-JEPA 2** (Meta/FAIR, 2025) -- 0.3B-1B self-supervised video encoder, MIT license, 284k+ downloads.
6. **VideoSearchR1** (June 2026) -- RL-enhanced video reasoning for search/retrieval, 2B params, updated 3 days ago.
7. **Find, Fix, Reason** (ICML 2026) -- Teacher-student RL for video reasoning.
8. **MSJoE** (CVPR 2026) -- MLLM-Sampler Joint Evolution for efficient long-form video understanding.

### 1.2 Keye-VL-2.0-30B-A3B [NEW] (June 2026, Kwai)

| Field | Value |
|-------|-------|
| Paper | arXiv:2606.10651 (June 9, 2026) |
| Architecture | MoE -- 30B total, only 3B activated per token |
| License | Apache 2.0 (same as Keye-VL 1.5) |
| Key innovation | First to adapt DeepSeek Sparse Attention (DSA) to GQA-based multimodal |
| Context | Lossless 256K context processing |
| Training | MOPD (Multi-Teacher On-Policy Distillation) + Context-RL + Video-RL |
| Benchmarks | SOTA among similar-scale models on TimeLens, Video-MME-v2, LongVideoBench |
| Agentic | Native agent collaboration across Code, Tool, Search |
| VRAM (est.) | ~6-8 GB (3B active + vision encoder) |
| Fits 12GB? | Likely yes |

**Why this matters:** The most architecturally novel model found. The MoE design
with 30B total / 3B active means it should fit comfortably on 12GB while having
the capacity of a much larger model. DSA for video is a first.

**Caveat:** Checkpoints were announced as released per paper (June 9). May require
gated access.

## 3. New Scene Detection Approaches

### 3.1 MSJoE [CVPR 2026]

| Field | Value |
|-------|-------|
| Paper | arXiv:2602.22932 (CVPR 2026) |
| Approach | RL-based key-frame sampler + MLLM jointly evolved |
| Impact | 8.0% accuracy gain over base MLLM |
| Datasets | VideoMME, LongVideoBench, LVBench, MLVU |

Not directly scene detection, but RL for optimal frame selection -- closely related
to the adaptive pipeline orchestration concept in v0.17.0 research.

### 3.2 PySceneDetect (Still Best Option)

No new alternatives discovered. The current approach (PySceneDetect with adaptive
threshold) remains the best open-source option for classical scene detection.

Emerging pattern: CLIP-based semantic scene change detection (embedding cosine
similarity between adjacent frames) is used as a complementary method.

## 8. Key Design Decisions

### Why V-JEPA 2 over alternatives?

| Alternative | Verdict |
|-------------|---------|
| InternVideo2-Stage1-1B (gated Apache 2.0) | Gated. V-JEPA 2 is open with no barriers. |
| VideoMAE v2 (Apache 2.0) | Smaller (86M) but less capable for video-level understanding. |
| Cosmos-Embed1 (NVIDIA Open) | Different use case (video-text retrieval). Complements V-JEPA 2. |

### Why Cosmos-Embed1 is worth trying

- Purpose-built for text-to-video retrieval (not frame-level, not image-text)
- Joint embedding space for both encoding directions
- 8-frame clip input captures temporal context
- Embedding dim (256-768) fits ChromaDB schema
- Fine-tuned anomaly detection variant available

### Why Keye-VL-2.0-30B-A3B needs investigation

- 3B active params from 30B MoE is the most parameter-efficient design found
- DSA for 256K lossless context is unique
- Apache 2.0 license
- Would be the most capable model that fits 12GB
